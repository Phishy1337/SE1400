Core HTML Structure, Semantics, Best Practices

Definition: HTML structures content using semantic elements that convey meaning to browsers, assistive tech, and search engines.

Use semantic tags to describe content purpose (navigation, main content, sections). Keep headings in logical order, use alt text on images, and ensure links and buttons are interactive elements.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Home</title>
</head>
<body>
  <header>Site Header</header>
  <nav><a href="#">Home</a> <a href="#">About</a></nav>
  <main>
    <article>
      <h1>Title</h1>
      <section>Section content</section>
    </article>
  </main>
  <aside>Sidebar</aside>
  <footer>Footer</footer>
</body>
</html>
  • Use landmark elements: header, nav, main, footer.
  • Heading order: h1 → h2 → h3 for nesting.
  • Links navigate, buttons act; don’t misuse spans/divs for interactivity.
  • Images need alt text that describes purpose; decorative images can use empty alt.

Metadata and Head Elements

Definition: The head contains metadata that configures the document and resources.

<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Page Title</title>
  <meta name="description" content="Summary of the page">
  <link rel="icon" href="/favicon.ico">
  <link rel="stylesheet" href="styles.css">
  <script src="app.js" defer></script>
</head>
  • doctype triggers standards mode.
  • lang improves accessibility and SEO.
  • viewport enables responsive layouts on mobile.

Block vs Inline vs Inline-Block

Definition: Elements participate in different formatting contexts affecting layout behavior.

Block: starts on new line, full width
Inline: flows within text, width by content
Inline-block: inline flow, but width/height apply

[Block]
-------------------------------+
|           DIV/P               |
-------------------------------+

[Inline]
Text A <span>B</span> C

[Inline-block]
Text A [button] [input] C

.block { display: block; }
.inline { display: inline; }
.inlineBlock { display: inline-block; }

Tables and Lists

Definition: Tables display tabular data; lists show ordered or unordered items.

<table>
  <caption>Scores</caption>
  <thead> <tr> <th scope="col">Name</th> <th scope="col">Score</th> </tr> </thead>
  <tbody> <tr> <th scope="row">A</th> <td>95</td> </tr> </tbody>
  <tfoot> <tr> <td colspan="2">Total</td> </tr> </tfoot>
</table>

<ol><li>Step 1</li><li>Step 2</li></ol>
<ul><li>Item</li><li>Item</li></ul>
<dl><dt>Term</dt><dd>Definition</dd></dl>

Media Elements

Definition: Built-in elements for images, audio, video with responsive and accessible features.

<img src="cat.jpg" alt="Cat" width="400">

<picture>
  <source media="(min-width: 800px)" srcset="hero-large.jpg">
  <source media="(min-width: 500px)" srcset="hero-med.jpg">
  <img src="hero-small.jpg" alt="Hero">
</picture>

<video src="movie.mp4" controls poster="poster.jpg" preload="metadata"></video>
<audio src="sound.mp3" controls preload="none"></audio>

Forms and Validation

Definition: Forms collect user input with built-in validation using attributes.

<form action="/submit" method="post" autocomplete="on">
  <fieldset>
    <legend>Contact</legend>
    <label for="name">Name</label>
    <input id="name" name="name" type="text" required minlength="2">

    <label for="email">Email</label>
    <input id="email" name="email" type="email" required>

    <label for="age">Age</label>
    <input id="age" name="age" type="number" min="0" max="120" step="1">

    <label>Plan</label>
    <input type="radio" name="plan" value="basic" required> Basic
    <input type="radio" name="plan" value="pro"> Pro

    <label for="terms">Accept</label>
    <input id="terms" type="checkbox" required>

    <label for="msg">Message</label>
    <textarea id="msg" name="msg" rows="4" placeholder="Hi"></textarea>
    <button type="submit">Send</button>
  </fieldset>
</form>
  • Associate labels with controls via for/id.
  • Use required, pattern, min, max, step.
  • Provide helpful errors via aria-describedby and visible text.

.card::before { content: ""; }
.card::after { content: ""; }

CSS Specificity

Definition: Determines which rule wins when multiple rules match.

Specificity weight: inline > ID > class/attr/pseudo-class > type/pseudo-element
Inline style: 1000
#id: 100
.class [attr] :hover :focus: 010
div p ::before: 001
!important overrides specificity, but use sparingly.

Box Model & Box-Sizing

Definition: Element size = content + padding + border; margins outside the box.

+----------- border -----------+
|   +------ padding ------+   |
|   |      content        |   |
|   +---------------------+   |
-----------------------------+
margin (outside)

.box { width: 200px; padding: 16px; border: 2px solid; margin: 12px; }
* { box-sizing: border-box; }

Use border-box to include padding/border in declared widths. Beware margin collapse between vertical margins of block elements.

Positioning

Definition: Controls how an element is placed relative to normal flow or a containing block.

.static { position: static; }
.relative { position: relative; top: 10px; }
.absolute { position: absolute; top: 10px; left: 10px; }
.fixed { position: fixed; bottom: 0; }
.sticky { position: sticky; top: 0; }

Absolutely positioned elements use the nearest ancestor with non-static position as reference. Sticky toggles between relative and fixed when crossing a threshold.

Display, Visibility, Overflow

Definition: Controls layout participation and how excess content is handled.

.hidden { display: none; }
.invisible { visibility: hidden; }
.transparent { opacity: 0; }
.scroll { overflow: auto; }
  • display: none removes from layout.
  • visibility: hidden keeps layout space.
  • opacity: 0 visible to hit-testing unless pointer-events disabled.

Flexbox

Definition: One-dimensional layout along a main axis with flexible sizing.

Main axis: row or column
Cross axis: perpendicular to main
Container: display:flex; flex-direction; flex-wrap; justify-content; align-items; align-content; gap
Items: order; flex-grow; flex-shrink; flex-basis; flex; align-self
.flex { display: flex; gap: 8px; justify-content: space-between; align-items: center; }
.item { flex: 1 1 200px; }

CSS Grid

Definition: Two-dimensional layout with explicit rows and columns.

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto 1fr auto;
  gap: 10px;
}
.item1 { grid-column: 1 / 3; }
.item2 { grid-row: 2; }
.item3 { grid-area: 3 / 1 / 4 / 4; }

Use fr units for proportional tracks, minmax() for responsive bounds, and named areas for template readability.

Typography, Units, Responsive Design

Definition: Control readable content with scalable units and breakpoints.

body { font-family: system-ui; font-size: 16px; line-height: 1.6; }
:root { font-size: 16px; }
.title { font-size: clamp(1.5rem, 2vw, 2.25rem); }
@media (max-width: 768px) { .grid { grid-template-columns: 1fr; } }
  • Units: px, em, rem, vw/vh, %, ch.
  • Prefer mobile-first: start narrow, add wider breakpoints.

Colors, Gradients, Backgrounds

Definition: Color systems and layered backgrounds for design.

.swatch { color: #333; background: hsl(220 10% 96%); }
.gradient { background: linear-gradient(90deg, #4f7cff, #84d18a); }
.bg { background: url(hero.jpg) center/cover no-repeat; }

Transitions, Transforms, Animations

Definition: Motion and state changes for interactive UI.

.btn { transition: transform 200ms ease, background 200ms ease; }
.btn:hover { transform: translateY(-2px) scale(1.02); }
@keyframes pulse { 0% { transform: scale(1); } 50% { transform: scale(1.05); } 100% { transform: scale(1); } }
.pulsing { animation: pulse 1.2s ease-in-out infinite; }

Common Pitfalls & Misconceptions

  • Margin collapse: adjacent vertical margins of blocks may combine; add padding or block formatting context to prevent.
  • Z-index needs a positioned element and respects stacking contexts created by transforms/opacity/positioning.
  • Absolute elements use nearest non-static ancestor as containing block.
  • Use flex for one-dimensional alignment, grid for two-dimensional layout.
  • Images should have explicit width/height to avoid layout shift; use object-fit for covering.
  • Minimize !important; refactor selectors and cascade instead.

Quick Reference Cheat Sheet

HTML

  • Skeleton: doctype, lang, meta charset, viewport, title
  • Semantics: header, nav, main, section, article, aside, footer
  • Forms: label+id, required/pattern/min/max
  • Media: picture+source for responsive images
  • Tables: thead/tbody/tfoot, th scope

CSS

  • Selectors: type, .class, #id, combinators, :hover/:focus
  • Specificity: inline > id > class > type
  • Box: border-box; margin/padding/border
  • Position: relative + absolute ancestor
  • Flexbox: flex container + flex items
  • Grid: template columns/rows, gap, place-items
  • Responsive: mobile-first, media queries
  • Motion: transition, transform, keyframes
